## Теоретическая часть

<details>
<summary>Основные понятия Git (нажмите, чтобы раскрыть)</summary>

- **commit (коммит)** — сохранение изменений в репозитории. Каждый коммит имеет уникальный хэш, автора, дату и сообщение, описывающее изменения.
- **branch (ветка)** — независимая линия разработки. Позволяет работать над новыми функциями или исправлениями, не влияя на стабильную версию кода (обычно в ветке `main` или `master`).
- **merge (слияние)** — операция объединения изменений из одной ветки в другую.
- **pull-request (запрос на слияние, PR)** — механизм предложения изменений из вашей ветки для включения их в целевую ветку репозитория. Позволяет обсуждать код до его объединения.
- **rebase (перебазирование)** — изменение "основы" ветки. Переносит коммиты текущей ветки на вершину другой ветки, создавая более линейную историю.
- **merge conflict (конфликт слияния)** — ситуация, когда Git не может автоматически объединить изменения из двух веток, так как они затрагивают одни и те же строки файла. Требует ручного разрешения.

</details>

<details>
<summary>Работа с локальным репозиторием и изменениями</summary>

- **`git init`** — создаёт новый локальный репозиторий в текущей директории.
- **`git status`** — показывает состояние рабочей директории: какие файлы изменены, добавлены или готовы к коммиту.
- **`git add <файл>`** — добавляет изменения указанного файла в следующую версию (в "staging area").
- **`git commit -m "сообщение"`** — сохраняет добавленные изменения в истории репозитория с описанием.
- **`git log --oneline`** — показывает сокращённую историю коммитов (хэш и сообщение).

</details>

<details>
<summary>Работа с удалённым репозиторием (GitHub)</summary>

- **`git remote add origin <url>`** — связывает локальный репозиторий с удалённым (на GitHub).
- **`git push origin <ветка>`** — отправляет локальные коммиты в удалённый репозиторий.
- **`git pull origin <ветка>`** — загружает изменения из удалённого репозитория и объединяет их с вашей локальной веткой.
- **`git fetch`** — загружает изменения из удалённого репозитория без автоматического слияния (позволяет сначала их просмотреть).

</details>

<details>
<summary>Работа с ветками</summary>

- **`git branch`** — показывает список локальных веток.
- **`git branch <название>`** — создаёт новую ветку (но не переключается на неё).
- **`git checkout -b <ветка>`** — создаёт новую ветку и сразу переключается на неё.
- **`git checkout <ветка>`** — переключается на существующую ветку.
- **`git branch -d <ветка>`** — удаляет локальную ветку.
- **`git push origin --delete <ветка>`** — удаляет ветку в удалённом репозитории.

</details>

<details>
<summary>Слияние и переписывание истории</summary>

- **`git merge <ветка>`** — объединяет изменения из указанной ветки в текущую.
- **`git rebase <ветка>`** — переносит коммиты текущей ветки на вершину другой ветки (переписывает историю).
- **`git rebase --continue`** — продолжает rebase после разрешения конфликтов.
- **`git push --force`** — принудительно отправляет изменения в удалённый репозиторий (используется после rebase).

</details>
